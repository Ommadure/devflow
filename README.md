# DevFlow

A productivity dashboard built with React, TypeScript, Vite, Supabase, and Tailwind CSS. The application provides user authentication, persisted notes, reusable code snippets, a Pomodoro-style timer, API testing utilities, GitHub analytics, and personalized settings.

## Project Overview

This project is a modern single-page application with:

- **Supabase authentication** for secure user login, signup, password reset, and magic links.
- **Supabase database storage** for user-specific data with row-level security (RLS).
- **A note-taking system** with Markdown preview and auto-saving.
- **A snippet manager** for storing and filtering reusable code snippets.
- **A focus timer** with Pomodoro-style sessions and persisted completed session counts.
- **Redux Toolkit + React Query** for state management and async data fetching.
- **Tailwind CSS** for responsive UI styling.

## Architecture

### Frontend

- `Vite` as the build tooling and development server.
- `React` with `TypeScript` for component-driven UI.
- `@reduxjs/toolkit` for global app state.
- `@tanstack/react-query` for server-state caching and query handling.
- `react-router-dom` for route-based navigation.
- `Tailwind CSS` + `tailwind-merge` for styling.
- `framer-motion` for animated UI transitions.
- `lucide-react` icons for UI elements.

### Backend / Database

- `Supabase` for authentication and PostgreSQL storage.
- `auth.users` for user identities.
- Custom tables: `snippets`, `notes`, `user_stats`.
- Row-level security policies to ensure users access only their own data.
- Trigger functions to maintain `updated_at` timestamps and initialize user statistics.

## Database Schema

The database schema is defined in `supabase/migrations/20260407000000_init_tables.sql`.

### `snippets`

- `id UUID PRIMARY KEY` generated by `gen_random_uuid()`
- `user_id UUID REFERENCES auth.users(id)`
- `title TEXT NOT NULL`
- `code TEXT NOT NULL`
- `language TEXT`
- `tags TEXT[] DEFAULT '{}'`
- `created_at TIMESTAMPTZ DEFAULT NOW()`
- `updated_at TIMESTAMPTZ DEFAULT NOW()`

Row-level security ensures that authenticated users can only select, insert, update, and delete their own snippets.

### `notes`

- `id UUID PRIMARY KEY` generated by `gen_random_uuid()`
- `user_id UUID REFERENCES auth.users(id)`
- `title TEXT NOT NULL`
- `content TEXT NOT NULL`
- `created_at TIMESTAMPTZ DEFAULT NOW()`
- `updated_at TIMESTAMPTZ DEFAULT NOW()`

Row-level security ensures that each user can manage only their own notes.

### `user_stats`

- `user_id UUID PRIMARY KEY REFERENCES auth.users(id)`
- `completed_sessions INTEGER DEFAULT 0`
- `updated_at TIMESTAMPTZ DEFAULT NOW()`

Row-level security ensures users can view and update only their own statistics.

### Trigger functions

- `update_modified_column()` updates `updated_at` on row updates for `snippets`, `notes`, and `user_stats`.
- `public.handle_new_user()` automatically inserts a `user_stats` row when a new user record is created in `auth.users`.

## Key Features

### Authentication

- Supports email/password sign in.
- Supports email sign up with password strength validation.
- Supports magic link sign in and password reset by email.
- Uses Supabase auth session persistence and auth state subscriptions.

### Code Snippets

- Create, update, delete, and list code snippets.
- Search by title or code content.
- Filter snippets by tags.
- Copy snippet code to clipboard.
- Sorts snippets by latest update.

### Notes

- Create notes with a title and Markdown content.
- Live Markdown preview using `react-markdown` and `remark-gfm`.
- Auto-save on content change after a debounce delay.
- Search notes by title or body text.
- Delete notes with confirmation.

### Focus Timer

- Pomodoro-inspired focus timer with default focus and break lengths.
- Start, pause, reset, and switch between focus and break modes.
- Persist completed focus sessions in Supabase.
- Reset session history with confirmation.

### Additional Pages

- `Dashboard` home page for summary and navigation.
- `GithubAnalytics` page for GitHub-related analytics.
- `ApiTesterPage` for testing API requests.
- `SettingsPage` for user preferences.

## Environment Setup

Create a `.env` file in the project root with these values:

```bash
VITE_SUPABASE_ANON_KEY=your-anon-public-key
```

> Make sure Supabase authentication is configured and the database schema is applied.
